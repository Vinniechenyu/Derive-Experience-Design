# 输出模板 / Output Patterns

只选择与交付物匹配的模板。双语输出时，每个中文段落后紧跟对应英文段落。  
Use only the pattern that matches the deliverable. For bilingual output, place the matching English section immediately after each Chinese section.

## 模板 D：作品集或汇报章节 / Pattern D: Portfolio or report chapter

```text
问题证据或背景 / Problem evidence or context
→ 根本体验问题 / Root experience problem
→ 体验目标 / Experience goals
→ 3–5 条体验策略 / 3–5 experience strategies
→ 用户旅程或页面顺序 / Journey or screen sequence
→ 设计图 + 设计点 + 收益 / Design image + design points + benefit
→ 验证或预期信号 / Validation or expected signal
```

每页只承担一个叙事任务。策略页讲“为什么”，方案页讲“如何在设计图中落地”。  
Give each page one narrative job. Strategy pages explain why; solution pages show how the strategy appears in the design.

方案页优先采用：/ For every solution page, prefer:

1. 设计图作为主要证据。/ Design image as primary evidence.
2. 三至四个锚定图片的设计点。/ Three or four design points anchored to the image.
3. 一个页底体验收益或验证目标。/ One bottom-line benefit or validation target.

## 有效表达句式 / Strong wording formulas

- “设计让用户从 **[旧行为]** 转向 **[新行为]**。” / “The design shifts the user from **[old behavior]** to **[new behavior]**.”
- “这不只是 **[视觉或布局选择]**，而是围绕 **[核心对象]** 重新组织 **[信息、控制或流程]**。” / “This is not merely **[visual or layout choice]**; it reorganizes **[information, control, or workflow]** around **[core object]**.”
- “默认视图保护 **[主线]**，同时通过 **[入口]** 保留对 **[细节]** 的访问。” / “The default view protects **[main concern]**, while **[entry point]** preserves access to **[detail]**.”
- “系统机制是 **[机制]**；体验价值在于让它变得 **[可见、可控、可恢复或可验证]**。” / “The system mechanism is **[mechanism]**; the experience value lies in making it **[visible, controllable, recoverable, or verifiable]**.”

避免 / Avoid:

- “布局更清晰。” / “The layout is clearer.”
- “提升效率。” / “It improves efficiency.”
- “更有沉浸感。” / “It is more immersive.”
- “提升用户体验。” / “It enhances user experience.”

用明确的行为、作用机制和收益替代泛化描述。  
Replace generic claims with a named behavior, mechanism, and benefit.
